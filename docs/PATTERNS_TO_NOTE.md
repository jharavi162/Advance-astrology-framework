## 📌 THE PROMOTION RULE (read this note)
> **A pattern graduates from this file into `AI_TRIANGULATION_PROMPT.md` (the
> director) ONLY after it has recurred — same signature, same direction — across
> ~10 independent analyses (different natives/charts).** Until a candidate hits
> that bar, it stays here as an observation and MUST NOT be treated as an
> established rule during analysis. One or two confirmations is anecdote; ~10 is
> a pattern worth hard-coding into the procedure.
>
> When a pattern reaches the bar: move it into the relevant Phase of the prompt,
> then mark it **PROMOTED** here (keep the row for history).

---

## How to use this log
- Each time you observe a candidate during an analysis, find its row and
  **increment the count**, adding a one-line citation (native + date + what
  confirmed/refuted it).
- If an analysis **refutes** a candidate, record that too (a miss counts as
  evidence — it may lower confidence or kill the candidate).
- Keep the verdict honest: log misses as openly as hits (no calibration).

Status legend: `CANDIDATE` (gathering evidence) · `PROMOTED` (now in the prompt) ·
`REJECTED` (failed enough times).
